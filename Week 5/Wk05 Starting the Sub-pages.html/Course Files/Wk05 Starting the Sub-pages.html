Step 01

For this assignment, you will be creating the two child pages in glitch.  You will perform the following tasks:   

  • give each page a proper and short name ending in .html.  The name of the page should not have punctuation or special characters and should be one or two words.  If for example, the page name that show in the nav bar can contain special characters such as: packages/rates.... but the name of the page should be just one of those, such as packages.html or rates.html
  • on the index.html page, fix your links to the child pages so they each have the correct name in the link to match the page name: example... if the name your child page is rivers.html, change the "#" so that it reads "/rivers.html" .. do this to each of the two page links (the / at the start of the name indicates to the browser that the page is within the website.
  • after you fix the page names on the index.html page so they are correct, copy and paste the complete header (from the <!DOCTYPE html> to just after the banner image on the index.html page and paste it to the top of each of your child pages... then do the same thing for the bottom of the index.html page and copy the complete footer down to the end of the html page and paste that code to  each of the two child pages after the top portion you just pasted.  Looking at each child page you will see the header and the footer with the center missing (the two parts will touch for now)
  • In the <title> of the child pages, make sure that where it says Home, you replace that with the correct name of the page such as || Packages and Rates or Rivers
  • If you did things correctly, you should be able to move from page to page using the nav bar links... make sure to check to be sure that this happens.

Want a shortcut? We know that many of the elements from our home page should also be on your child pages...header, navigation, footer, colors, etc. So use the index.html as a starting point. Make a new file in your editor and save it with a meaningful name (for example if I were making a contact page I would name my new file contact.html)...then copy and paste the code from your index.html into your new file, update the <title>, and delete the content that is specific to the home page and Voila! you have a great start on your new page.

Note

You will not create .css pages for your new pages.... all of the css  will be on the style.css.  Having multiple .css files  could create a problem of having pages look different in important ways because the styles in one file were updated and not the other. So, we will have only have one .css file for your entire site. Add any new styles for this new html page to the bottom of your existing .css.

Add a comment showing where items for the child page 2 and child page 3 will go (of course, the items that are already declared and used on the home page should not be written again ... example, on the home page you used sec1, sec2, and sec3... on the rivers page, you use the same layout format and also use sec1, sec2, and sec3 so do not put them a second time in the .css for the rivers page, just new items.

On your style.css page, write a comment such as: /*  Items for Rivers page below here */ and put the rivers specific css items there.... skip some lines and put a comment for your other child page.  Setting your page up like this will make it easier to stay organized and to spot errors when things do not work right.

Always check your page by looking at in your browser to make sure that your links work correctly, that your images all display correctly, that your page has no errors and has the required images and text and has good visual appeal.

What is meant by “visually appealing?”


I wish there was a non-subjective way to measure this... but there isn't. Generally you only lose points for this with things that are obvious. For example, text that is difficult to read because it uses a color to similar to the background, a navigation bar that is directly against a banner or the text of the page rather than having some space around it, use of a color scheme that obviously indicates no thought was put into the color choices (this doesn't mean that you can't use white - it's more stating that if you just threw colors onto the page for the sake of throwing colors onto the page and it is obvious, you'll lose points), etc.

If you simply copy/pasted the text from the content page in most cases it will not be visually appealing.  Make sure that you format it to make it more readable! Be generous with <p> tags, padding, margins, and maybe even <ul>s.  You will lose points on this if you do not put some thought into the readability of the text.

What is considered “significant content?”

Each page of a website should clearly convey to the user what the page is for. If it is a main/home page, it should provide the user with enough information to know what your site is about, whether it contains the information/products they are looking for, and, thereby, whether they should stick around and keep looking through your website. As another example, if you have a "Contact Us" page, you usually need more than just a contact form/email address. The page should have enough information that the user knows what the contact form/email address is for, what happens when they contact you, what type of information you want the user to provide when contacting you, etc.
